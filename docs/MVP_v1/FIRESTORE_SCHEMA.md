## 🔒 Firestore Security Rules (MVP v1)

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    
    // Allow authenticated users to read their own data
    match /users/{userId} {
      allow read: if request.auth.uid == userId;
      allow create: if request.auth.uid == userId;
      allow update: if request.auth.uid == userId;
    }
    
    match /user_profiles/{userId} {
      allow read: if request.auth.uid == userId;
      allow create: if request.auth.uid == userId;
      allow update: if request.auth.uid == userId;
    }
    
    match /quiz_sessions/{gameId} {
      allow read: if request.auth.uid == resource.data.userId;
      allow create: if request.auth.uid != null;
      allow update: if request.auth.uid == resource.data.userId;
    }
    
    match /difficulty_progress/{userId} {
      allow read: if request.auth.uid == userId;
      allow create: if request.auth.uid == userId;
      allow update: if request.auth.uid == userId;
    }
    
    // Public read for questions (to avoid calling functions for every question)
    match /questions/{questionId} {
      allow read: if request.auth.uid != null;
    }
    
    // Only Cloud Functions can write to ai_logs
    match /ai_logs/{logId} {
      allow read: if request.auth.uid != null && request.auth.uid == resource.data.userId;
    }
  }
}
```

## 🎯 Data Flow (MVP v1)

```
1. User Signs Up
   ↓
   Create /users/{uid}
   Create /user_profiles/{uid}
   Create /difficulty_progress/{uid}

2. User Starts Quiz
   ↓
   Create /quiz_sessions/{gameId}
   GET random question from /questions

3. User Answers Questions
   ↓
   Update /quiz_sessions/{gameId} with answers
   Log to /ai_logs if needed

4. User Completes Quiz
   ↓
   Finalize /quiz_sessions/{gameId}
   Update /user_profiles/{uid}
   Update /difficulty_progress/{uid}
```
